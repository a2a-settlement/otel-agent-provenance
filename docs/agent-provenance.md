# Semantic Conventions for Agent Output Provenance

**Status:** Development

## Overview

These conventions define attributes for capturing the provenance of agent outputs across three attestation tiers. They answer the question: "Where did this output come from, and can the claim be verified?"

The conventions extend above the existing `gen_ai.*` agent conventions, which define agent identity and invocation patterns, by adding source tracking, integrity verification, and live attestation capabilities.

## Three-Tier Model

### Tier 1 -- Self-Declared

Any agent can emit these attributes without infrastructure. The agent declares its own provenance; no external verification.

### Tier 2 -- Signed / Hash-Verified

The agent provides cryptographic evidence: content hashes and signatures that can be validated. An attestation record may exist on a verification backend.

### Tier 3 -- Verifiable Source Callback

A live endpoint can confirm the provenance claim in real-time. This is the highest assurance level, requiring verification infrastructure.

## Attributes

### Identity

| Attribute | Type | Requirement Level | Description | Examples |
|-----------|------|-------------------|-------------|----------|
| `agent.id` | string | Required | Unique agent identifier | `agent-abc123` |
| `agent.identity.tier` | int | Conditional | KYA verification tier (1-3) | `1`, `2`, `3` |
| `agent.identity.registry` | string | Recommended | Identity registry | `a2a-settlement.org` |

### Output Provenance (Tier 1)

| Attribute | Type | Requirement Level | Description | Examples |
|-----------|------|-------------------|-------------|----------|
| `agent.output.provenance.tier` | int | Required | Attestation tier (1/2/3) | `1` |
| `agent.output.source.type` | enum | Required | How output was produced | `retrieval`, `model_generation` |
| `agent.output.source.uri` | string[] | Recommended | Source URIs that influenced output | `["https://example.com"]` |
| `agent.output.source.influence` | enum | Recommended | How sources influenced output | `attended`, `cited`, `ignored` |
| `agent.output.confidence` | double | Recommended | Calibrated confidence (0.0-1.0) | `0.85` |
| `agent.output.model.name` | string | Recommended | Model that produced output | `gpt-4o` |
| `agent.output.model.version` | string | Recommended | Model version | `2025-05-01` |

### Grounding Metrics

| Attribute | Type | Requirement Level | Description | Examples |
|-----------|------|-------------------|-------------|----------|
| `agent.output.grounding.coverage` | double | Recommended | Fraction of output grounded by sources | `0.85` |
| `agent.output.grounding.source_count` | int | Recommended | Number of distinct sources | `3` |
| `agent.output.grounding.domain_count` | int | Recommended | Number of distinct domains | `2` |

### Content Integrity (Tier 2)

| Attribute | Type | Requirement Level | Description |
|-----------|------|-------------------|-------------|
| `agent.output.hash.algorithm` | enum | Conditional (tier >= 2) | `sha256`, `sha3-256`, `sha384`, `sha512` |
| `agent.output.hash.value` | string | Conditional (tier >= 2) | Hex-encoded hash |
| `agent.output.signature.method` | enum | Conditional (tier >= 2) | `hmac`, `ed25519`, `x509` |
| `agent.output.signature.value` | string | Conditional | Base64 signature |
| `agent.output.signature.key_id` | string | Conditional | Signing key reference |

### Attestation (Tier 2/3)

| Attribute | Type | Requirement Level | Description |
|-----------|------|-------------------|-------------|
| `agent.provenance.attestation.uri` | string | Conditional (tier >= 2) | Attestation record URI |
| `agent.provenance.attestation.timestamp` | string | Conditional (tier >= 2) | RFC 3339 timestamp |

### Verifiable Callback (Tier 3)

| Attribute | Type | Requirement Level | Description |
|-----------|------|-------------------|-------------|
| `agent.provenance.callback.uri` | string | Conditional (tier = 3) | Live verification endpoint |
| `agent.provenance.callback.method` | enum | Conditional | `GET` or `POST` |
| `agent.provenance.callback.status` | enum | Recommended | `verified`, `expired`, `revoked` |

### Provenance Chain

| Attribute | Type | Requirement Level | Description |
|-----------|------|-------------------|-------------|
| `agent.provenance.chain.parent_span_id` | string | Recommended | Parent agent's output span ID |
| `agent.provenance.chain.depth` | int | Recommended | Hops from root task |
| `agent.provenance.chain.root_task_id` | string | Recommended | Original task ID |

## Source Type Values

| Value | Description |
|-------|-------------|
| `model_generation` | Output generated by a model |
| `retrieval` | Output derived from retrieved documents |
| `tool_call` | Output produced by a tool invocation |
| `agent_delegation` | Output produced by a delegated agent |
| `hybrid` | Output from multiple source types |

## Source Influence Values

| Value | Description |
|-------|-------------|
| `attended` | Source was in the model context window and influenced output |
| `cited` | Source is explicitly cited in the output |
| `ignored` | Source was retrieved but not used |

## Relationship to `gen_ai.*` Conventions

These conventions sit above the existing `gen_ai.*` namespace:

- `gen_ai.agent.id` identifies the agent; `agent.id` may be the same value
- `gen_ai.request.model` identifies the model for a call; `agent.output.model.name` identifies the model that produced the final output
- `gen_ai.evaluation.score.value` is a basic score; `agent.task.acceptance_criteria.*` provides the full evaluation protocol
- `gen_ai.data_source.id` identifies a data source; `agent.output.source.uri` identifies specific sources within that data source

Instrumentations SHOULD set both `gen_ai.*` and `agent.*` attributes where applicable.
